# 曜舞天堂 AI 工作規範

這份文件是給任何接手曜舞天堂專案的 AI 使用。
目標是避免資料遺失、風格跑掉、改完沒有紀錄。

## 每次開始工作前

新的 AI 或新的聊天室必須先讀：

1. `AI_HANDOFF_曜舞天堂.md`
2. `PROJECT_MEMORY.md`
3. `AI_WORKFLOW.md`
4. 若是影片或文宣任務，再讀：
   - `影片處理skill/README.md`
   - `yaowu-remotion-video/src/Composition.tsx`

## 每次修改時

- 先確認使用者需求是網站、內容、圖片、影片、後台、文案，還是部署。
- 改網站時，只動本次任務相關檔案。
- 改內容時，所有外來 HTML 都要轉成曜舞天堂黑金風格。
- 改影片時，影片與素材預設屬於行銷後端資料，不推上網站 Git。
- 改表單或後台時，不要提交 secret、token、密碼、管理密語。
- 若發現其他 AI 或使用者已改過檔案，先讀懂差異，不要直接覆蓋。

## 每次完成後必須記錄

所有 AI 每次完成實作後，都要更新至少一份紀錄：

- 一般網站、內容、功能更新：更新 `CHANGELOG.md`
- 長期重要設定、外部服務、部署方式、核心風格：同步更新 `PROJECT_MEMORY.md`
- 跨 AI 交接會需要知道的規則或注意事項：同步更新 `AI_HANDOFF_曜舞天堂.md`
- 影片製作、剪輯流程、輸出檔、素材來源：同步更新 `AI_HANDOFF_曜舞天堂.md` 的影片段落，必要時也更新 `影片處理skill/README.md`

## CHANGELOG 寫法

每次新增紀錄時，把新區塊加在 `CHANGELOG.md` 的標記上方：

```markdown
<!-- 新紀錄請在此處上方新增 -->
```

格式：

```markdown
## YYYY-MM-DD HH:mm - 簡短標題

### 類型
- 網站 / 內容 / 影片 / 圖片 / 後台 / 表單 / 部署 / 文件

### 異動
- 說明實際做了什麼。
- 說明影響範圍。

### 相關檔案
- `path/to/file`

### 驗證
- 有跑測試就寫測試。
- 沒跑測試就寫未執行與原因。

### 備註
- 是否已推 Git。
- 是否只是本地行銷素材。
```

## PROJECT_MEMORY 更新時機

符合以下任一條件就要更新 `PROJECT_MEMORY.md`：

- 改了官方網站、LINE、贊助、推文回報等外部連結。
- 改了伺服器設定倍率、版本名稱、開服時間。
- 改了後台、Google Apps Script、Cloudflare Worker、部署方式。
- 新增或隱藏重要頁面。
- 建立新的長期流程，例如影片製作流程、文宣模板、AI 交接規範。

## AI_HANDOFF 更新時機

符合以下任一條件就要更新 `AI_HANDOFF_曜舞天堂.md`：

- 使用者偏好有變。
- 視覺風格規則有變。
- 新增重要素材位置。
- 新增重要影片輸出。
- 新增需要新 AI 特別避開的陷阱。
- 新增部署或 Git 注意事項。

## 回覆使用者時

完成任務後要簡短說明：

- 改了什麼。
- 紀錄更新在哪裡。
- 是否有推 Git。
- 是否有未執行的測試或未完成事項。

## 禁止事項

- 不要未經明確要求把行銷影片、素材資料夾推到網站 Git。
- 不要提交 OAuth secret、token、密碼、管理密語。
- 不要用白底文件風格直接貼進前台頁面。
- 不要改與任務無關的大量檔案。
- 不要把其他天堂名稱保留在公開內容中，除非是使用者刻意指定。
